# Regra 15 — Internacionalização (I18n)

## Objetivo
Template **universal** com suporte multi-idioma usando `@angular/localize` + JSON translation files. Suporte inicial para **PT-BR** (padrão) e **EN-US**.

---

## Stack I18n

| Ferramenta | Função |
|------------|--------|
| `@angular/localize` | API oficial Angular para i18n |
| JSON translation files | Arquivos de tradução em `src/i18n/{locale}.json` |
| `TranslocoService` | Service customizado para runtime i18n (alternativa ao build-time) |
| Browser locale detection | Auto-detecção de idioma do navegador |

---

## Estratégia Obrigatória

### 1. Runtime I18n (Recomendado)
- Usar JSON files para traduções (`pt-BR.json`, `en-US.json`)
- Idioma alterável em runtime (sem rebuild)
- Persistir preferência no `localStorage`

### 2. Build-time I18n (Alternativa)
- Usar `@angular/localize` com `$localize`
- Build separado por locale (`ng build --localize`)
- Deploy de múltiplos bundles

**DECISÃO:** Usar **runtime i18n** para flexibilidade e UX superior.

## Configuração

### src/i18n/i18n.config.ts

```typescript
export interface Locale {
  code: string;           // ISO 639-1 + ISO 3166-1
  label: string;          // Nome nativo do idioma
  flag: string;           // Emoji da bandeira
}

export const AVAILABLE_LOCALES: Locale[] = [
  { code: 'pt-BR', label: 'Português', flag: '🇧🇷' },
  { code: 'en-US', label: 'English', flag: '🇺🇸' }
];

export const DEFAULT_LOCALE = 'pt-BR';

export function getBrowserLocale(): string {
  const browserLang = navigator.language || 'pt-BR';
  
  // Normalizar para formato correto
  const normalized = browserLang.replace('_', '-');
  
  // Retornar se locale disponível
  const match = AVAILABLE_LOCALES.find(
    loc => loc.code === normalized || loc.code.startsWith(normalized.split('-')[0])
  );
  
  return match?.code ?? DEFAULT_LOCALE;
}
```

## Service de Tradução

### core/i18n/i18n.service.ts

```typescript
import { Injectable, signal, computed, inject } from '@angular/core';
import { HttpClient } from '@angular/common/http';
import { AVAILABLE_LOCALES, DEFAULT_LOCALE, getBrowserLocale } from '../../i18n/i18n.config';

type Translations = Record<string, unknown>;

@Injectable({ providedIn: 'root' })
export class I18nService {
  private readonly http = inject(HttpClient);
  
  private readonly _currentLocale = signal<string>(this.getStoredLocale());
  private readonly _translations = signal<Translations>({});
  private readonly _loading = signal(false);

  readonly currentLocale = this._currentLocale.asReadonly();
  readonly translations = this._translations.asReadonly();
  readonly loading = this._loading.asReadonly();
  
  readonly availableLocales = AVAILABLE_LOCALES;

  constructor() {
    this.loadTranslations(this._currentLocale());
  }

  /**
   * Traduz uma chave com interpolação de variáveis.
   * Exemplo: translate('errors.minLength', { min: 3 }) → "Mínimo de 3 caracteres"
   */
  translate(key: string, params?: Record<string, string | number>): string {
    const keys = key.split('.');
    let value: unknown = this._translations();

    for (const k of keys) {
      if (value && typeof value === 'object' && k in value) {
        value = (value as Record<string, unknown>)[k];
      } else {
        console.warn(`[I18n] Translation key not found: ${key}`);
        return key; // Fallback: retorna a própria key
      }
    }

    if (typeof value !== 'string') {
      console.warn(`[I18n] Translation value is not a string: ${key}`);
      return key;
    }

    // Interpolação de variáveis
    if (params) {
      return value.replace(/\{\{(\w+)\}\}/g, (_, param) => String(params[param] ?? ''));
    }

    return value;
  }

  /**
   * Muda o idioma atual e recarrega traduções.
   */
  setLocale(locale: string): void {
    if (!AVAILABLE_LOCALES.some(l => l.code === locale)) {
      console.error(`[I18n] Locale not available: ${locale}`);
      return;
    }

    this._currentLocale.set(locale);
    localStorage.setItem('locale', locale);
    this.loadTranslations(locale);
  }

  private loadTranslations(locale: string): void {
    this._loading.set(true);

    this.http.get<Translations>(`/i18n/${locale}.json`).subscribe({
      next: (translations) => {
        this._translations.set(translations);
        this._loading.set(false);
        document.documentElement.lang = locale;
      },
      error: (err) => {
        console.error(`[I18n] Failed to load translations for ${locale}:`, err);
        this._loading.set(false);
      }
    });
  }

  private getStoredLocale(): string {
    const stored = localStorage.getItem('locale');
    if (stored && AVAILABLE_LOCALES.some(l => l.code === stored)) {
      return stored;
    }
    return getBrowserLocale();
  }
}
```

## Pipe de Tradução

### core/i18n/translate.pipe.ts

```typescript
import { Pipe, PipeTransform, inject, computed, Signal } from '@angular/core';
import { I18nService } from './i18n.service';

@Pipe({
  name: 'translate',
  standalone: true,
  pure: false // Re-executa quando locale muda
})
export class TranslatePipe implements PipeTransform {
  private readonly i18n = inject(I18nService);

  transform(key: string, params?: Record<string, string | number>): string {
    return this.i18n.translate(key, params);
  }
}
```

## Interceptor (Accept-Language)

### core/i18n/i18n.interceptor.ts

```typescript
import { HttpInterceptorFn } from '@angular/common/http';
import { inject } from '@angular/core';
import { I18nService } from './i18n.service';

/**
 * Adiciona header Accept-Language em todas as requisições.
 */
export const i18nInterceptor: HttpInterceptorFn = (req, next) => {
  const i18n = inject(I18nService);
  
  const cloned = req.clone({
    setHeaders: {
      'Accept-Language': i18n.currentLocale()
    }
  });

  return next(cloned);
};
```

## Uso em Templates

### ✅ SEMPRE usar o pipe translate

```html
<!-- Tradução simples -->
<button>{{ 'common.save' | translate }}</button>

<!-- Tradução com parâmetros -->
<p>{{ 'errors.minLength' | translate:{ min: 3 } }}</p>

<!-- Tradução em atributos -->
<input [placeholder]="'auth.email' | translate" />

<!-- Tradução em títulos -->
<h1>{{ 'products.title' | translate }}</h1>
```

### ✅ Uso em componentes (TypeScript)

```typescript
import { Component, inject, computed } from '@angular/core';
import { I18nService } from '@/core/i18n/i18n.service';

@Component({
  selector: 'app-my-component',
  standalone: true,
  template: `<p>{{ errorMessage() }}</p>`
})
export class MyComponent {
  private readonly i18n = inject(I18nService);

  readonly errorMessage = computed(() => 
    this.i18n.translate('errors.unexpected', { correlationId: '123' })
  );
}
```

```typescript
// language-selector.component.ts
import { Component, inject } from '@angular/core';

@Component({

}
```

## Antipadrões Proibidos

### ❌ Texto hardcoded no template

```html
<!-- ❌ PROIBIDO -->
<button>Salvar</button>
<p>Campo obrigatório</p>

<!-- ✅ CORRETO -->
<button>{{ 'common.save' | translate }}</button>
<p>{{ 'errors.required' | translate }}</p>
```

### ❌ Tradução inline no TypeScript

```typescript
// ❌ PROIBIDO
const message = this.i18n.currentLocale() === 'pt-BR' ? 'Salvo' : 'Saved';

// ✅ CORRETO
const message = this.i18n.translate('common.saved');
```

### ❌ JSON translation keys sem namespace

```json
// ❌ PROIBIDO - chaves globais poluem namespace
{
  "save": "Salvar",
  "delete": "Excluir"
}

// ✅ CORRETO - namespaces por contexto
{
  "common": {
    "save": "Salvar",
    "delete": "Excluir"
  }
}
```

## Convenções de Nomenclatura

### Translation Keys

| Contexto | Namespace | Exemplo |
|----------|-----------|---------|
| Botões comuns | `common.*` | `common.save`, `common.cancel` |
| Autenticação | `auth.*` | `auth.login`, `auth.logout` |
| Erros de validação | `errors.*` | `errors.required`, `errors.email` |
| Feature específica | `{feature}.*` | `products.title`, `users.createUser` |
| Mensagens de sucesso | `{feature}.{action}` | `products.created`, `users.updated` |

### Interpolação de Variáveis

Usar `{{variableName}}` no JSON:

```json
{
  "greeting": "Olá, {{name}}!",
  "itemsCount": "{{count}} itens encontrados"
}
```

```typescript
i18n.translate('greeting', { name: 'João' });        // "Olá, João!"
i18n.translate('itemsCount', { count: 42 });         // "42 itens encontrados"
```

## Integração com Backend

### Header Accept-Language

O interceptor envia automaticamente:

```http
GET /api/v1/products HTTP/1.1
Accept-Language: pt-BR
Authorization: Bearer {token}
X-Tenant: acme
```

Backend pode usar para:
- Retornar mensagens de erro traduzidas
- Formatar datas/números conforme locale
- Ordenar resultados alfabeticamente (collation)

### ProblemDetails Traduzido

Se backend retornar `problem.detail` em inglês, sobrescrever com tradução local:

```typescript
// products.store.ts
if (apiError.status >= 500) {
  const detail = this.i18n.translate('errors.unexpected', { 
    correlationId: apiError.correlationId 
  });
  this.error.set(detail);
} else {
  this.error.set(apiError.problem.detail ?? 'Unknown error');
}
```

## Testes

### Service

```typescript
import { TestBed } from '@angular/core/testing';
import { provideHttpClient } from '@angular/common/http';
import { HttpTestingController, provideHttpClientTesting } from '@angular/common/http/testing';
import { I18nService } from './i18n.service';

describe('I18nService', () => {
  let service: I18nService;
  let httpMock: HttpTestingController;

  beforeEach(() => {
    TestBed.configureTestingModule({
      providers: [provideHttpClient(), provideHttpClientTesting()]
    });
    service = TestBed.inject(I18nService);
    httpMock = TestBed.inject(HttpTestingController);
  });

  it('deve traduzir chave simples', () => {
    const req = httpMock.expectOne('/i18n/pt-BR.json');
    req.flush({ common: { save: 'Salvar' } });

    expect(service.translate('common.save')).toBe('Salvar');
  });

  it('deve traduzir com interpolação', () => {
    const req = httpMock.expectOne('/i18n/pt-BR.json');
    req.flush({ errors: { minLength: 'Mínimo de {{min}} caracteres' } });

    expect(service.translate('errors.minLength', { min: 3 }))
      .toBe('Mínimo de 3 caracteres');
  });

  it('deve mudar idioma', () => {
    service.setLocale('en-US');
    expect(service.currentLocale()).toBe('en-US');
    expect(localStorage.getItem('locale')).toBe('en-US');
  });
});
```

## Checklist I18n

Antes de considerar uma feature completa, validar:

- [ ] Nenhum texto hardcoded no template
- [ ] Todas as strings usam `{{ 'key' | translate }}`
- [ ] Translation keys organizados por namespace
- [ ] JSON files `pt-BR.json` e `en-US.json` atualizados
- [ ] Interpolação de variáveis funcional
- [ ] Mensagens de erro do backend traduzidas localmente
- [ ] Seletor de idioma visível no layout
- [ ] Locale persistido no `localStorage`
- [ ] Header `Accept-Language` enviado ao backend
- [ ] Testes do `I18nService` passando
